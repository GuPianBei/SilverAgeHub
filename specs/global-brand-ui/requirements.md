# 银发纪全局品牌 UI 优化需求

## 范围

本轮仅优化现有微信小程序页面的 WXML/WXSS 与视觉素材引用，重点覆盖用户端首页、教程、课程、预约和“我的”主链路；管理端只统一必要的可用性与品牌色。

## 用户故事

1. 作为银发用户，我希望首页直接看到教程推荐，并能用大按钮进入教程、课程和预约。
2. 作为银发用户，我希望所有页面的文字、颜色、卡片和按钮保持一致，减少重新理解界面的负担。
3. 作为运营人员，我希望首页顶部预留课程新闻轮播区域，后续可由研发部接入真实数据。

## 验收标准

1. 当用户打开首页时，小程序应显示可扩展的 `swiper` 顶部区域，当前使用静态品牌内容。
2. 当用户浏览首页主体时，小程序应将“常用服务”替换为“教程推荐”，展示多篇独立教程，并由每张推荐卡直接进入对应教程详情。
3. 当用户进入教程、课程、预约和“我的”页面时，小程序应统一使用品牌蓝、暖米白、深青文字和一致的卡片层级。
4. 当页面显示加载、空状态或错误状态时，小程序应保持不低于 32rpx 的可读字号和明确的文字说明。
5. 当用户操作主要按钮时，小程序应提供不低于 88rpx 的触控高度。
6. 在本轮修改期间，小程序不得改变现有事件绑定、字段、接口、导航目标、`userApi`、`adminApi` 或云函数。

## 非目标

- 不接入真实课程新闻数据。
- 不实现基于用户或热度的动态教程推荐。
- 不新增页面、功能、Web 管理后台或自定义 tabBar。
- 不修改 JS、service、云函数、数据库字段和路由配置。
