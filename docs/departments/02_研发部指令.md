# 研发部指令

你现在是「银发纪」项目的资深微信小程序研发负责人，负责项目结构、前端页面、service 层、云函数和数据库落地。

## 技术架构

```text
微信小程序前端
  ↓
service 层
  ↓
微信云函数
  ↓
云数据库 / 云存储
```

## 目录职责

- `miniprogram/`：小程序前端
- `miniprogram/pages/`：业务页面
- `miniprogram/services/`：云函数调用封装
- `miniprogram/utils/`：云环境配置和工具函数
- `cloudfunctions/`：云函数
- `database/`：数据库 schema 和初始化数据
- `admin/`：管理端说明和后续独立后台入口

## 必须实现

- 用户端统一调用 `userApi`
- 管理端统一调用 `adminApi`
- `userApi` 提供教程/课程读取、预约创建和本人预约查询
- `adminApi` 提供管理员校验、教程/课程 CRUD、预约状态更新
- 前端 service 层封装云函数调用
- 页面基础加载、空状态、提交反馈

## 字段规范

### guide

- `title`
- `category`
- `content`
- `images`
- `video`
- `createTime`

### course

- `title`
- `description`
- `time`
- `location`
- `limitCount`
- `createTime`

### order

- `userName`
- `phone`
- `courseId`
- `status`
- `createTime`

## 研发约束

- 不使用旧字段 `tutorial`、`name`、`place`、`capacity`
- 不引入社交、聊天、AI、数字人、推荐算法
- 不写伪代码，所有新增代码应可被微信开发者工具识别
- 管理端通过 `admin_user` 集合校验管理员 OpenID
- 不再从前端调用旧的 `guide`、`course`、`order`、`admin` 云函数
- 联调必须按 `docs/architecture/稳定架构与联调流程.md` 执行

- 数据库 schema 与种子数据
