# 银发纪 Codex 三部门协作总则

本项目将 Codex 工作流拆分为产品部、研发部、UI 部。三部门围绕同一个 MVP 目标协作：完成一个可运行、可演示、可继续扩展的「银发纪」微信小程序。

## 协作目标

- 产品部负责定义做什么、为什么做、做到什么程度
- 研发部负责决定怎么实现、如何保证代码可运行
- UI 部负责保证适老化体验、页面清晰度和视觉一致性

## 统一产品边界

MVP 只做：

- 教程学习
- 课程展示
- 课程预约
- 我的预约
- 管理端基础能力

MVP 不做：

- 社交系统
- 实时聊天
- AI 生成内容
- 数字人
- 复杂推荐算法

## 统一字段口径

- 教程集合：`guide`
- 课程集合：`course`
- 预约集合：`order`
- 预约人字段：`userName`
- 课程地点字段：`location`
- 课程人数上限字段：`limitCount`
- 预约状态：`未确认`、`已确认`、`完成`

## 交接顺序

1. 产品部输出需求范围、页面清单、字段口径、验收标准
2. UI 部输出页面结构、适老化规范、组件与状态建议
3. 研发部按产品和 UI 口径实现页面、服务层、云函数和数据库
4. 三部门共同检查：功能是否完整、体验是否清晰、代码是否可运行

验收基准统一使用 `docs/departments/04_MVP验收清单.md`。

## 输出格式要求

每个部门输出时都要包含：

- 本轮目标
- 已确认内容
- 待处理问题
- 下一步交给哪个部门
