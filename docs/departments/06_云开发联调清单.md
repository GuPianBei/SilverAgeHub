# 云开发联调清单

原始联调日期：2026-06-16

> 本清单已于 2026-06-18 按稳定架构更新。详细故障定位见 `docs/architecture/稳定架构与联调流程.md`。

## 1. 项目配置

- 在 `project.config.json` 中将 `appid` 从 `touristappid` 替换为真实微信小程序 AppID。
- 在 `miniprogram/utils/config.js` 中填写真实云开发环境 ID `envId`。
- 使用微信开发者工具打开项目根目录 `D:\作业素材\银发纪`。

## 2. 云数据库集合

在微信云开发控制台创建以下集合：

- `guide`
- `course`
- `order`
- `admin_user`

字段口径以 `database/*.schema.json` 为准：

- `guide`：`title`、`category`、`content`、`images`、`video`、`createTime`
- `course`：`title`、`description`、`time`、`location`、`limitCount`、`createTime`
- `order`：`userName`、`phone`、`courseId`、`status`、`createTime`

## 3. 种子数据

- 可参考 `database/seed.json` 手动导入 `guide` 和 `course` 初始数据。
- `order` 不建议导入种子数据，预约应通过小程序表单真实提交生成。

## 4. 云函数部署

在微信开发者工具中只上传并部署：

- `cloudfunctions/userApi`
- `cloudfunctions/adminApi`

部署后测试：

- `userApi`：教程/课程列表与详情、预约创建、本人预约列表
- `adminApi`：身份检查、教程/课程 CRUD、预约状态更新

## 5. 完整链路联调

1. 打开首页，确认显示「银发纪」。
2. 进入教程列表，打开一篇教程详情。
3. 进入课程列表，打开课程详情，确认展示 `limitCount`。
4. 点击报名，填写 `userName` 和 `phone`。
5. 提交预约，确认 `order` 集合写入记录，默认 `status` 为 `未确认`。
6. 进入我的预约，确认可以看到刚提交记录。
7. 在 `admin_user` 录入管理员 OpenID。
8. 进入管理分包，将预约状态更新为 `已确认`。
9. 回到我的预约，确认状态变化。

## 6. 上线前风险

- 管理端已接入 `admin_user` 白名单校验，必须完成管理员初始化。
- 数据库权限需按生产规则收紧，避免普通用户直接写管理数据。
- 真实环境需要确认云函数服务端权限可访问 `guide`、`course`、`order` 集合。
