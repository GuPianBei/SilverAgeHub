# 银发纪 MVP 验收清单

本清单用于判断「银发纪」微信小程序 MVP 是否达到可演示、可联调、可继续开发的状态。验收时按页面、业务链路、数据、云函数、UI 适老化和项目资料逐项检查。

## 一、验收范围

MVP 必须包含：

- 首页
- 教程列表
- 教程详情
- 课程列表
- 课程详情
- 预约表单
- 我的预约
- 管理端基础入口
- 云函数与数据库基础能力

MVP 不验收：

- 社交系统
- 实时聊天
- AI 生成内容
- 数字人
- 复杂推荐算法
- 完整权限后台

## 二、用户端验收

### 首页

- 页面能正常打开，无白屏、无明显报错
- 第一屏能看出产品名称「银发纪」
- 有清晰入口进入教程模块
- 有清晰入口进入课程模块
- 页面文案适合老年用户理解

### 教程列表

- 能调用 `guide` 数据并展示教程列表
- 每条教程至少展示标题 `title`
- 每条教程能展示分类 `category`
- 点击教程能进入教程详情页
- 无数据时有空状态提示

### 教程详情

- 能根据教程 ID 获取详情
- 展示教程标题 `title`
- 展示教程分类 `category`
- 展示教程内容 `content`
- 预留图片 `images` 和视频 `video` 字段的展示扩展空间
- 返回上一页流程正常

### 课程列表

- 能调用 `course` 数据并展示课程列表
- 每条课程至少展示课程名称 `title`
- 展示课程时间 `time`
- 展示课程地点 `location`
- 点击课程能进入课程详情页
- 无数据时有空状态提示

### 课程详情

- 能根据课程 ID 获取详情
- 展示课程名称 `title`
- 展示课程介绍 `description`
- 展示课程时间 `time`
- 展示课程地点 `location`
- 展示人数上限 `limitCount`
- 有明确的报名入口

### 预约表单

- 能从课程详情进入预约表单
- 表单包含预约人姓名字段 `userName`
- 表单包含联系电话字段 `phone`
- 提交前校验姓名和电话不能为空
- 提交成功后写入 `order` 集合
- 新预约默认状态为 `未确认`
- 提交后有明确成功反馈

### 我的预约

- 能展示当前用户的预约记录
- 每条预约能展示关联课程或课程 ID
- 每条预约能展示状态 `status`
- 状态文案只使用 `未确认`、`已确认`、`完成`
- 无预约时有空状态提示

## 三、管理端验收

### 管理入口

- 管理端页面能正常打开
- 页面包含教程管理入口
- 页面包含课程管理入口
- 页面包含预约管理入口
- 管理端文案面向助教/运营人员，不面向普通长者

### 教程管理

- 支持查看教程列表
- 支持新增教程的能力或已预留入口
- 支持编辑教程的能力或已预留入口
- 支持删除教程的能力或已预留入口

### 课程管理

- 支持查看课程列表
- 支持创建课程的能力或已预留入口
- 支持维护课程时间、地点、人数上限
- 支持发布或展示课程

### 预约管理

- 支持查看预约列表
- 能看到预约人 `userName`
- 能看到联系电话 `phone`
- 能看到关联课程 `courseId`
- 能更新预约状态或已预留入口

## 四、云函数验收

### userApi 用户端云函数

- 能返回教程列表和教程详情
- 能返回课程列表和课程详情
- 用户端不能新增、修改或删除教程和课程
- 能创建预约记录
- 创建预约时写入 `userName`、`phone`、`courseId`
- 创建预约时默认 `status` 为 `未确认`
- 能只返回当前用户的预约记录
- 返回结构统一为 `{ code, message, data }`

### adminApi 管理端云函数

- 能检查当前 OpenID 是否存在于 `admin_user`
- 能按资源查询 `guide`、`course`、`order`
- 能更新教程、课程或预约状态
- 能删除教程或课程
- 未授权账号返回 `403`
- 对未知资源返回明确错误
- 返回结构统一为 `{ code, message, data }`

## 五、数据库验收

### guide 集合

- 字段包含 `title`
- 字段包含 `category`
- 字段包含 `content`
- 字段预留 `images`
- 字段预留 `video`
- 字段包含 `createTime`

### course 集合

- 字段包含 `title`
- 字段包含 `description`
- 字段包含 `time`
- 字段包含 `location`
- 字段包含 `limitCount`
- 字段包含 `createTime`

### order 集合

- 字段包含 `userName`
- 字段包含 `phone`
- 字段包含 `courseId`
- 字段包含 `status`
- 字段包含 `createTime`
- `status` 只使用 `未确认`、`已确认`、`完成`

### admin_user 集合

- 字段包含 `openid`
- 字段包含布尔值 `enabled`
- 只有 `enabled = true` 的账号可以执行管理操作

## 六、UI 适老化验收

- 正文字号不低于 `32rpx`
- 主要按钮高度不低于 `88rpx`
- 页面入口清晰，不依赖复杂说明
- 表单字段数量少，填写负担低
- 文字与背景对比清晰
- 点击区域足够大，避免误触
- 空状态提示清楚
- 成功、失败、加载状态有反馈
- 页面文案避免技术词和复杂表达

## 八、完整链路验收

验收时至少跑通以下链路：

1. 打开首页
2. 进入教程列表
3. 打开一篇教程详情
4. 返回首页或切换到课程列表
5. 打开课程详情
6. 点击报名
7. 填写姓名和电话
8. 提交预约
9. 进入我的预约
10. 查看刚提交的预约状态

链路通过标准：

- 全流程无白屏
- 无阻断性报错
- 数据能写入数据库
- 页面能读出数据库数据
- 老年用户能理解每一步要做什么
