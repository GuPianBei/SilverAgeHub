## 本轮目标

将项目切换到用户端/管理端隔离的新架构，并跑通真实云环境。

## 禁止事项

- 不再修改或部署旧 `guide`、`course`、`order`、`admin` 作为正式入口
- 不把管理页面移回 `miniprogram/pages`
- 不再使用 `ADMIN_OPENIDS` 环境变量
- 不修改统一响应结构 `{ code, message, data }`
- 未完成前一项时不要跳到下一项

## 执行步骤

### 1. 打开项目

使用微信开发者工具打开项目根目录，确认：

- AppID：`wx9f9b638a412469b0`
- envId：`cloud1-d0g5eg4kxeed0d74d`

如果实际环境不同，只修改 `miniprogram/utils/config.js`，不要在多个文件重复填写。

### 2. 创建集合

确认存在：

- `guide`
- `course`
- `order`
- `admin_user`

### 3. 部署 userApi

右键 `cloudfunctions/userApi`：

1. 上传并部署
2. 选择云端安装依赖
3. 云端测试输入 `{ "action": "health" }`
4. 确认版本为 `2026-06-18-stable-v1`

版本不一致时停止后续操作，重新部署。

### 4. 部署 adminApi

右键 `cloudfunctions/adminApi`：

1. 上传并部署
2. 选择云端安装依赖
3. 云端测试输入 `{ "action": "whoami" }`
4. 确认 `code = 0`
5. 记录返回的 `openid`
6. 确认版本为 `2026-06-18-stable-v1`

### 5. 初始化管理员

在 `admin_user` 新增：

```json
{
  "openid": "上一步返回的 openid",
  "enabled": true,
  "role": "admin",
  "name": "管理员"
}
```

再次测试 `adminApi` 的 `whoami`，必须返回 `isAdmin = true`。

### 6. 用户端测试

按顺序检查：

1. 教程列表
2. 课程列表
3. 我的预约

允许显示空状态，不允许显示超时或云函数错误。

### 7. 管理端测试

从「我的」进入运营管理：

1. 新增教程
2. 新增课程
3. 编辑教程
4. 编辑课程
5. 删除测试教程或课程

### 8. 预约闭环

1. 用户端打开课程
2. 提交预约
3. 我的预约显示 `未确认`
4. 管理端改为 `已确认`
5. 用户端下拉刷新后显示 `已确认`
