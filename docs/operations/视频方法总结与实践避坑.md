# 视频方法总结与实践避坑

整理日期：2026-06-20

## 视频核心方法

视频展示的是一种 AI 辅助开发后台的落地方式：

1. 使用腾讯云 CloudBase 承担云数据库、云函数、云存储等基础设施。
2. 先建立数据库模型，再迁移或补齐数据。
3. 不直接把 CloudBase CMS 当作最终用户界面，因为其表单化界面可用但不够贴合业务。
4. 使用 Vue 3 管理后台框架，并使用 Element Plus 等成熟组件库完成后台界面。
5. 让 AI 按阶段生成数据库脚本、后台框架和业务页面。
6. 每完成一个阶段就测试，确认后再进入下一阶段。

本质上是：

```text
CloudBase 提供后端能力
成熟后台框架提供界面骨架
组件库提供视觉与交互基础
AI 负责连接、生成和迭代
人工负责规则、验收和纠偏
```

## 对银发纪的应用

银发纪继续使用当前微信云开发架构，不重新购买服务器或域名。

- 用户端：微信小程序主包
- 管理端：当前先使用小程序管理分包
- 用户接口：`userApi`
- 管理接口：`adminApi`
- 数据库：`guide`、`course`、`order`、`admin_user`

如果后期管理端业务明显变复杂，再单独建立 Vue 3 + Element Plus Web 管理后台。MVP 阶段不要同时维护小程序管理端和 Web 管理端。

## 实践避坑

### 不让 AI 一次完成全部系统

正确拆分：

1. 数据结构
2. 云函数协议
3. 用户端链路
4. 管理端链路
5. 权限
6. 联调
7. UI 优化

每步必须有可验证结果。

### 不同时维护两套正式接口

银发纪正式接口只有：

- `userApi`
- `adminApi`

旧 `guide`、`course`、`order`、`admin` 只作历史兼容，不允许继续作为新开发入口。

### 不让 AI 自行改变架构

涉及以下内容必须先由总控确认：

- 新增框架
- 新增数据库集合
- 修改字段名
- 修改云函数入口
- 用户端与管理端合并或拆分
- 引入 Web 管理后台

### 不用聊天记录充当项目文档

聊天记录会过期、截断和相互污染。最终决定必须写入项目文件，窗口只读取最新版文件。

### 不跳过真实环境验证

本地代码正确不代表云端已更新。修改云函数后必须重新部署，并使用版本探针确认云端版本。

### 不允许多个窗口同时修改同一模块

同一时间一个文件只能有一个责任窗口。其他窗口只提供建议，不直接改该文件。
