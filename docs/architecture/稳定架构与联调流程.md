# 银发纪稳定架构与联调流程

## 架构结论

项目采用「一个微信小程序项目、两个前端域、两个云函数入口」。

```text
用户主包 miniprogram/pages
  └─ userApi
      ├─ guide：列表、详情
      ├─ course：列表、详情
      └─ order：创建、查询本人预约

管理分包 miniprogram/admin
  └─ adminApi
      ├─ 管理员身份检查
      ├─ guide：增删改查
      ├─ course：增删改查
      └─ order：列表、状态更新
```

两端共享同一云环境，以及 `guide`、`course`、`order`、`admin_user` 集合。

## 首次联调固定流程

必须严格按顺序执行。

### 1. 确认配置

- `project.config.json` 的 `appid` 是当前小程序 AppID
- `miniprogram/utils/config.js` 的 `envId` 是当前云环境 ID
- 开发者工具当前选择的云环境与 `envId` 完全一致

### 2. 创建集合

创建：

- `guide`
- `course`
- `order`
- `admin_user`

业务访问统一经过云函数，联调阶段先不要配置复杂数据库权限。

### 3. 部署函数

只部署：

1. `userApi`
2. `adminApi`

选择云端安装依赖。部署完成后重新编译小程序。

### 4. 验证用户端

依次打开教程列表、课程列表、我的预约。三页能显示数据或空状态，且没有报错，即表示 `userApi` 已连通。

### 5. 初始化管理员

1. 在「我的」点击「进入运营管理」
2. 记录未授权页面显示的 OpenID
3. 在 `admin_user` 集合新增：

```json
{
  "openid": "页面显示的 OpenID",
  "enabled": true,
  "role": "admin",
  "name": "管理员"
}
```

4. 返回管理页下拉刷新

注意：`enabled` 必须是布尔值 `true`，不能是字符串 `"true"`。

### 6. 录入测试数据

使用管理端新增一条教程和一门课程，然后返回用户端检查数据。

### 7. 跑通预约闭环

1. 用户端打开课程详情并报名
2. 提交姓名和电话
3. 在我的预约确认状态为 `未确认`
4. 管理端将状态改为 `已确认`
5. 用户端下拉刷新确认状态变化

## 故障定位

### 一直加载或超时

- 检查对应 API 是否已部署
- 检查部署环境是否与 `envId` 一致
- 检查修改云函数后是否重新上传
- 查看云函数日志是否提示集合不存在

### 提示返回格式不正确

云端仍是旧版本，重新部署 `userApi` 或 `adminApi`。

### 管理端没有权限

- 核对 `admin_user.openid`
- 核对 `enabled` 为布尔值 `true`
- 核对集合与云函数位于同一环境

## 发布前设置

联调结束后将 `miniprogram/utils/config.js` 中的：

```js
enableAdminEntry: false
```

普通用户界面将不再显示管理入口。
