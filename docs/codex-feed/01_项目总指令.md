# 银发纪 Codex 项目总指令

你现在参与「银发纪」项目开发。项目采用三部门协作方式推进：产品部、研发部、UI 部。执行任务前，需要先确认自己当前扮演的部门角色，并遵守 `docs/departments/` 中对应部门指令。

银发纪面向老年群体，核心目标是解决数字技能学习和线下课程预约问题。项目必须围绕「教程学习 + 课程预约 + 内容管理」闭环展开。

## 部门分工

- 产品部：定义需求范围、页面流程、业务字段、验收标准
- 研发部：实现小程序页面、service 层、云函数、数据库
- UI 部：把控适老化界面、交互路径、字号按钮和视觉一致性

## 必做模块

- 教程系统：教程列表、分类筛选、教程详情，支持图文步骤与视频字段
- 课程系统：课程列表、课程详情，展示时间、地点、人数上限
- 预约系统：填写姓名和电话，提交课程预约，查看预约状态
- 管理端：教程管理、课程管理、预约管理

## 技术要求

- 前端使用微信小程序原生页面结构
- 后端使用微信云函数
- 数据使用云数据库集合 `guide`、`course`、`order`
- 图片和视频使用云存储字段预留
- 代码需要可运行，不写伪代码

## 工作方式

1. 产品部先确认本轮要做的功能和验收标准
2. UI 部给出页面结构和适老化体验要求
3. 研发部按产品与 UI 口径实现代码
4. 完成后回到产品部进行范围和验收检查

## 禁止范围

- 不做社交系统
- 不做实时聊天
- 不做 AI 生成内容
- 不做数字人
- 不做复杂推荐算法
