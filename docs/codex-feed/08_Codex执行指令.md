# Codex 执行指令

请基于微信小程序 + 云开发生成「银发纪」MVP，要求代码可直接在微信开发者工具中打开并继续调试。

执行前先判断当前任务属于哪个部门：

- 需求、范围、流程、验收：交给产品部
- 页面样式、交互、适老化检查：交给 UI 部
- 代码、云函数、数据库、调试：交给研发部

部门指令统一存放在 `docs/departments/`。

## 必须实现

1. 教程模块：列表 + 详情
2. 课程模块：列表 + 详情
3. 预约模块：表单提交 + 云数据库写入 + 我的预约
4. 管理端：教程、课程、预约的基础管理入口
5. 云函数：用户端统一使用 `userApi`，管理端统一使用 `adminApi`
6. service 层：封装前端对云函数的调用
7. 数据库说明：提供 `guide`、`course`、`order`、`admin_user` 字段设计

## 架构约束

- 用户页面必须位于主包 `miniprogram/pages`
- 管理页面必须位于分包 `miniprogram/admin`
- 用户端不得调用教程或课程写接口
- 管理端操作必须经过 `adminApi` 的 OpenID 校验
- 所有云函数返回统一的 `{ code, message, data }` 结构
- 联调只部署 `userApi` 和 `adminApi`

## 字段统一

- 教程集合使用 `guide`，不要再使用 `tutorial`
- 预约人字段使用 `userName`，不要使用 `name`
- 课程地点字段使用 `location`，不要使用 `place`
- 课程人数上限字段使用 `limitCount`，不要使用 `capacity`
- 预约状态使用 `未确认`、`已确认`、`完成`

## 开发边界

禁止实现社交、实时聊天、AI 生成内容、数字人、复杂推荐算法。所有功能必须可解释、可落地、可演示。
